HOW IT WORKS

From raw folders to session-ready playlists

  1. STEP I

    Point at your library

    Add as many local folders as you like from the sidebar. The scanner indexes them by MD5 hash so duplicates never slip through.

  2. STEP II

    Tag by mood

    Drag tracks into moods — Combat, Stealth, Tavern, Boss — or build character OSTs for recurring NPCs and monsters.

  3. STEP III

    Download missing pieces

    Paste a Spotify album URL or a YouTube playlist link. spotdl and yt-dlp handle the rest; a live progress bar tracks each download.

  4. STEP IV

    Play during your session

    Queue tracks from any mood, shuffle, loop, trim intros with the GPU-accelerated clip editor, and your volume setting persists across reloads.

FEATURES

Everything your table needs

Library Scanner

Recursively indexes multiple root folders. MD5 deduplication catches duplicates across different paths.

Mood Tagging

Assign Combat, Stealth, Tavern, Dungeon, Boss, and custom moods. Build per-character OSTs for recurring NPCs.

In-Browser Player

Queue, shuffle, loop. Volume level and mute state persist in localStorage across browser reloads.

Download Queue

Paste a Spotify or YouTube URL. Downloads run in the background via SSE live-progress and survive server restarts.

Audio Trim Editor

Drag start/end handles to trim intros and outros. Exported via ffmpeg with GPU hardware acceleration when available.

Organize Library

Dry-run preview shows exactly which files move where before you commit. Copies or moves into mood-based folder trees.

TECH STACK

No bundler. No framework. Just Node.

Single-file server, single-page frontend, built-in SQLite. Install it, run it, own it — no build step, no lock-in.
